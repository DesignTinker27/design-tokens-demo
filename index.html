Red Flags - Don't Do This

  • ❌ **Don't use primitive colors directly**: Always use semantic or interactive tokens in your designs and code. Primitive colors are for defining the system, not direct application.
  • ❌ **Don't hardcode spacing values**: Rely on `--spacing-*` tokens for all layout and element spacing to maintain consistency.
  • ❌ **Don't mix semantic meanings**: Ensure the color and context align (e.g., don't use a success color for an error state).

Decision Trees

Choosing Colors

  • ✅ **Are you styling an interactive element (button, link, input state)?**
    • ➡️ Use `interactive-*` tokens (e.g., `--interactive-primary`, `--interactive-destructive-hover`).
  • ✅ **Are you indicating a status or meaning (success, warning, error, info)?**
    • ➡️ Use `semantic-*` tokens (e.g., `--semantic-success`, `--semantic-error`).
  • ✅ **Are you styling a background, card, or border?**
    • ➡️ Use `surface-*` tokens (e.g., `--surface-background`, `--surface-card`).
  • ✅ **Are you styling text?**
    • ➡️ Use `text-*` tokens (e.g., `--text-primary`, `--text-muted`).
  • ❌ If none of the above, consult the Design System team. Avoid using primitive colors directly.
